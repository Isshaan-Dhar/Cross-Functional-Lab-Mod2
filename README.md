# Git Workflow Lab Assignment

## Objective
This repository serves as the practical execution environment for the Git Workflow Lab. The purpose of this assignment is to demonstrate proficiency in fundamental version control operations, including branching strategies, remote repository synchronization, and the Merge Request (MR) lifecycle.

## Lab Requirements & Execution Steps
To complete this lab, the following Git workflow was executed:

1. **Clone Repository:** Cloned the remote repository to the local environment.
2. **Create Feature Branch:** Branched off `main` to isolate development changes.
3. **Modify Code:** Updated project files (including this README).
4. **Commit Changes:** Staged and committed modifications with descriptive commit messages.
5. **Push Branch:** Pushed the local feature branch to the remote origin.
6. **Create Merge Request:** Opened an MR to merge the feature branch into `main`.
7. **Resolve Comments:** Addressed reviewer feedback and pushed subsequent commits to resolve the MR threads.
8. **Merge Branch:** Successfully merged the approved feature branch into the `main` branch.
